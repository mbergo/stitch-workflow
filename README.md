# Stitch Design Workflow — MCP Server

**Prompt → UI → Code → Your Editor. One pipeline.**

An MCP server that orchestrates Google Stitch's AI-powered UI generation into a seamless design-to-code pipeline for **Claude Code**, **VS Code**, and **Google Antigravity**.

```
┌──────────────┐     ┌──────────────┐     ┌─────────────────────────┐
│  Your Prompt │────▶│ Google Stitch │────▶│  This MCP Server        │
│              │     │ (Gemini 3)   │     │  ┌───────────────────┐  │
│ "dark crypto │     │              │     │  │ generate_screen   │  │
│  dashboard"  │     │  Renders UI  │     │  │ extract_design    │  │
│              │     │  HTML/CSS    │     │  │ export_to_devtool │  │
└──────────────┘     └──────────────┘     │  │ full_pipeline     │  │
                                          │  └───────────────────┘  │
                                          └──────────┬──────────────┘
                                                     │
                              ┌───────────────────────┼───────────────────────┐
                              ▼                       ▼                       ▼
                     ┌──────────────┐        ┌──────────────┐        ┌──────────────┐
                     │  Claude Code │        │    VS Code   │        │ Antigravity  │
                     │              │        │              │        │              │
                     │ DESIGN.md    │        │ .vscode/     │        │ .antigravity/│
                     │ .claude/     │        │ src/         │        │ rules.md     │
                     │ src/         │        │ DESIGN.md    │        │ src/         │
                     │ components/  │        │ components/  │        │ DESIGN.md    │
                     └──────────────┘        └──────────────┘        └──────────────┘
```

## What It Does

1. **`stitch_generate_screen`** — Send a natural language prompt, get a rendered UI screen
2. **`stitch_extract_design`** — Pull design tokens (colors, fonts, spacing) from any screen
3. **`stitch_export_to_devtool`** — Package designs for your editor with proper config files
4. **`stitch_full_pipeline`** — One call does everything: generate → extract → export

Each export target gets **tool-specific configuration**:
- **Claude Code**: `.claude/settings.json` + `DESIGN.md` (auto-read by Claude)
- **VS Code**: `.vscode/settings.json` + Tailwind config
- **Antigravity**: `.antigravity/rules.md` + agent instructions
- **Local Files**: Raw HTML/CSS + screenshots

## Prerequisites

- **Node.js 18+**
- **Google Cloud account** with Stitch API enabled
- **One of**: Stitch API Key, OAuth token, or gcloud ADC configured

### Get a Stitch API Key (Easiest)

1. Go to [stitch.withgoogle.com](https://stitch.withgoogle.com)
2. Click your profile icon → **Stitch Settings**
3. Go to **API Keys** → **Create Key**
4. Copy the key

### Or Use gcloud ADC

```bash
gcloud auth login
gcloud auth application-default login
gcloud config set project YOUR_PROJECT_ID
gcloud beta services mcp enable stitch.googleapis.com
```

## Setup Per Tool

### Claude Code

```bash
# One-liner setup
claude mcp add stitch-workflow -e STITCH_API_KEY=your-key -- node /path/to/stitch-workflow/dist/server.js
```

Or add to `~/.claude/settings.json`:
```json
{
  "mcpServers": {
    "stitch-workflow": {
      "command": "node",
      "args": ["/path/to/stitch-workflow/dist/server.js"],
      "env": { "STITCH_API_KEY": "your-key" }
    }
  }
}
```

**Usage in Claude Code:**
```
> claude "Use stitch_full_pipeline to build a dark crypto dashboard and export to claude_code"
```

Claude Code will auto-read the generated `DESIGN.md` and use the design tokens for all subsequent code generation.

---

### VS Code

Add to `.vscode/mcp.json` in your project:
```json
{
  "mcpServers": {
    "stitch-workflow": {
      "command": "node",
      "args": ["/path/to/stitch-workflow/dist/server.js"],
      "env": { "STITCH_API_KEY": "your-key" }
    }
  }
}
```

Works with **Cline**, **Continue**, **Copilot Chat**, or any VS Code extension that supports MCP.

---

### Google Antigravity

Add via Settings → MCP → Add New Server, or in `~/.antigravity/mcp.json`:
```json
{
  "mcpServers": {
    "stitch-workflow": {
      "command": "node",
      "args": ["/path/to/stitch-workflow/dist/server.js"],
      "env": { "STITCH_API_KEY": "your-key" }
    }
  }
}
```

**Antigravity workflow:**
1. The agent reads `.antigravity/rules.md` (generated by the export)
2. Reads `DESIGN.md` for design tokens
3. Builds React components from the Stitch HTML
4. Tests in the built-in browser
5. Iterates until it matches the screenshots

## Tools Reference

### `stitch_generate_screen`

Generate a UI screen from a text prompt.

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `prompt` | string | ✅ | What to build (min 10 chars) |
| `projectId` | string | | Existing project, or auto-creates |
| `platform` | "web" \| "mobile" | | Default: "web" |
| `style` | string | | "dark theme", "glassmorphism", etc. |
| `designSystemUrl` | string | | URL to extract design system from |

### `stitch_extract_design`

Extract design tokens from a generated screen.

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `projectId` | string | ✅ | Stitch project ID |
| `screenId` | string | ✅ | Screen ID |

Returns: fonts, colors, spacing, components, layoutType.

### `stitch_export_to_devtool`

Export designs with tool-specific config.

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `projectId` | string | ✅ | Stitch project ID |
| `screenIds` | string[] | ✅ | Screens to export |
| `target` | string | ✅ | claude_code, vscode, antigravity, local_files |
| `outputDir` | string | | Custom output path |
| `includeDesignMd` | boolean | | Default: true |
| `includeScreenshots` | boolean | | Default: true |

### `stitch_full_pipeline`

End-to-end: prompt → generate → extract → export.

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `prompt` | string | ✅ | What to build |
| `targets` | string[] | ✅ | Export targets |
| `platform` | string | | "web" or "mobile" |
| `style` | string | | Visual style |
| `variants` | number | | Design variants to generate (1-6) |
| `outputDir` | string | | Custom output path |

### `stitch_health_check`

Verify Stitch API connectivity and authentication.

### `stitch_list_projects` / `stitch_list_screens` / `stitch_get_screen_code`

Direct access to Stitch project and screen data.

## Architecture

```
src/
├── server.ts                  # MCP server entry — tool registration & routing
├── tools/
│   ├── generate-screen.ts     # stitch_generate_screen
│   ├── extract-design.ts      # stitch_extract_design
│   ├── export-to-devtool.ts   # stitch_export_to_devtool (the bridge)
│   └── full-pipeline.ts       # stitch_full_pipeline (orchestrator)
├── prompts/
│   └── templates.ts           # AI prompt constants (design analysis, code conversion)
├── connectors/
│   └── stitch-client.ts       # Stitch API client (auth, retry, logging)
├── validators/
│   └── schemas.ts             # Zod schemas for all inputs/outputs
├── utils/
│   ├── logger.ts              # Structured logging
│   ├── retry.ts               # Exponential backoff
│   └── health-check.ts        # Service reachability
└── configs/
    └── mcp-configs.json       # Config examples for all targets
```

### Design Decisions

- **Zod validation on every boundary** — no unvalidated data enters or leaves
- **Structured logging** — every tool call: timestamp, tool, duration, hash, pass/fail
- **Retry with jitter** — all external calls get 3 retries with exponential backoff
- **Graceful degradation** — if one variant fails, others continue; if one export target fails, others continue
- **DESIGN.md as the bridge format** — this is the standard that Stitch, Antigravity, and Claude Code all understand
